# Загрузочные носители

ISO-образы для установки и загрузки операционных систем в виртуальных машинах.

## Обзор

Раздел «Загрузочные носители» (Настройки → Загрузочные носители) отображает все ISO-образы в настроенных директориях. Используйте их при создании или редактировании ВМ для загрузки с CD-ROM.

## Директории

По умолчанию приложение сканирует:

- `/var/lib/qemu/iso`
- `/srv/iso`

Настройте пути в `.env`:

```
QEMU_ISO_DIRECTORIES=/var/lib/qemu/iso,/srv/iso,/путь/к/кастомной
```

## Добавление ISO-образов

**Вариант 1: Загрузка через веб-интерфейс**

Если хотя бы одна директория из `QEMU_ISO_DIRECTORIES` доступна для записи, отображается форма загрузки:

1. Перейдите в Настройки → Загрузочные носители
2. Выберите целевую директорию (если доступно несколько)
3. Нажмите «Обзор» и выберите ISO-файл
4. Дождитесь завершения загрузки (прогресс показывает стадию: загрузка во временную папку → перемещение в целевую)

**Лимиты:** Файлы до 10 ГБ (при наличии свободного места в staging и целевой директориях). PHP и Nginx настраиваются на 10 ГБ через `install.sh`.

**Ошибки:** Сбои загрузки (недостаточно места, ошибка перемещения и т.п.) записываются в Настройки → Логи → вкладка «Ошибки».

**Вариант 2: Копирование вручную**

1. Скопируйте ISO-файлы в одну из директорий (например, `/var/lib/qemu/iso/` или `/srv/iso/`)
2. Нажмите «Обновить список» или перезагрузите страницу для отображения новых файлов

При создании ВМ выберите директорию и файл в выпадающем списке.

## Скачивание

Нажмите иконку скачивания, чтобы сохранить ISO на компьютер. Поддерживается докачка (HTTP Range).

## Удаление

Нажмите иконку удаления, чтобы удалить ISO. Удаление выполняет **QemuBootImagesControlService** (C++ gRPC/HTTP сервис) на хосте. Сервис должен быть запущен до удаления файлов через веб-интерфейс.

**Важно:** C++ сервис работает на хосте. Для Docker в `.env` переменная `BOOT_MEDIA_SERVICE_URL` должна указывать на хост:

- Host network: `http://127.0.0.1:50052`
- `http://host.docker.internal:50052` (рекомендуется, работает на Linux и Docker Desktop)
- `http://172.17.0.1:50052` (шлюз Docker bridge)

`install.sh` задаёт это автоматически. Сервис запускается и останавливается через `start.sh` и `stop.sh`.

Если подключение из Docker не работает (cURL error 7), выполните:

```bash
sudo ./scripts/fix-boot-media-docker.sh
sudo docker compose restart app
```

Если сервис недоступен, приложение использует прямой `unlink` или ставит запрос в очередь. Обработка очереди:

```bash
php artisan boot-media:process-deletions
```

Или добавьте в crontab: `* * * * * /путь/к/QemuWebControl/scripts/process-boot-media-deletions.sh`

## Docker

При работе в Docker директории ISO монтируются с хоста. Убедитесь, что пути на хосте существуют и содержат ISO-файлы:

```yaml
volumes:
  - ${QEMU_ISO_VOLUME:-/var/lib/qemu/iso}:/var/lib/qemu/iso:ro
  - /srv/iso:/srv/iso:ro
```

По умолчанию директории только для чтения (`:ro`). Для загрузки через веб-интерфейс смонтируйте хотя бы одну директорию с правами записи (уберите `:ro`).
