## Полезные советы

### Использование значений по умолчанию

Для быстрой установки с настройками по умолчанию просто нажимайте Enter:

```bash
./install.sh --lang ru

# Выбор БД: Enter (Docker MariaDB)
# Apache2: n Enter (не настраивать)
# Готово!
```

### Изменение только одного параметра

Если нужно изменить только один параметр (например, имя БД):

```bash
Введите хост базы данных [localhost]: ⏎
Введите порт базы данных [3306]: ⏎
Введите имя базы данных [qemu_control]: my_custom_db  ← изменили
Введите имя пользователя БД [qemu_user]: ⏎
```

### Автоматическая установка (без интерактива)

Для автоматической установки можно предварительно настроить `.env`:

```bash
# 1. Скопируйте и настройте .env
cp .env.example .env
nano .env

# 2. Установите нужные значения
DB_HOST=localhost
DB_PORT=3306
DB_DATABASE=qemu_control
DB_USERNAME=qemu_user
DB_PASSWORD=mypassword

# 3. Запустите установку
# Скрипт использует значения из .env как значения по умолчанию
./install.sh --lang ru
```

## Примечания

- **[значение]** - значение по умолчанию, используется при нажатии Enter
- **⏎** - нажатие клавиши Enter (использовать значение по умолчанию)
- **✓** - успешное выполнение
- **✗** - ошибка
- **⚠** - предупреждение
- **➜** - процесс выполняется

## Что делать если...

### Ошибка при вводе

Если вы ввели неправильное значение:
1. Дождитесь завершения установки
2. Отредактируйте `.env`
3. Перезапустите контейнеры: `./restart.sh`

### Хотите изменить настройки после установки

```bash
# 1. Остановите контейнеры
./stop.sh

# 2. Отредактируйте .env
nano .env

# 3. Если изменили БД, запустите миграции
docker compose exec app php artisan migrate:fresh --seed

# 4. Запустите контейнеры
./start.sh
```

### Забыли пароль БД

Если вы забыли пароль, который ввели:

```bash
# Посмотрите в .env
cat .env | grep DB_PASSWORD
```

**Важно:** `.env` содержит пароли в открытом виде, будьте осторожны!
